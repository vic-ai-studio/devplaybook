# Building a Documentation Site with Docusaurus 3 and Algolia Search 2026

Docusaurus (by Meta) is the most feature-complete documentation site generator for technical projects. It handles versioning, i18n, search, and custom React components out of the box. Version 3.x brought React 18, MDX 3, and improved performance. This guide covers a production-ready setup.

## Creating a New Docusaurus 3 Site

```bash
npx create-docusaurus@latest my-docs classic --typescript
cd my-docs
npm run start
```

The `classic` preset gives you: docs (versioning, sidebars), blog (changelog), and a landing page.

## Core Configuration

```typescript
// docusaurus.config.ts
import { themes as prismThemes } from 'prism-react-renderer';
import type { Config } from '@docusaurus/types';

const config: Config = {
  title: 'My Project',
  tagline: 'The fastest way to build X',
  favicon: 'img/favicon.ico',
  url: 'https://docs.myproject.com',
  baseUrl: '/',
  organizationName: 'myorg',
  projectName: 'my-project',

  onBrokenLinks: 'throw',  // Fail build on broken links
  onBrokenMarkdownLinks: 'warn',

  i18n: {
    defaultLocale: 'en',
    locales: ['en', 'ja'],
  },

  presets: [
    ['classic', {
      docs: {
        sidebarPath: './sidebars.ts',
        editUrl: 'https://github.com/myorg/my-project/tree/main/docs/',
        showLastUpdateAuthor: true,
        showLastUpdateTime: true,
        versions: {
          current: { label: '3.x (Next)' },
        },
      },
      blog: {
        showReadingTime: true,
        blogTitle: 'Changelog',
        blogDescription: 'Release notes and updates',
      },
      theme: {
        customCss: './src/css/custom.css',
      },
    }],
  ],

  themeConfig: {
    navbar: {
      title: 'My Project',
      logo: { alt: 'Logo', src: 'img/logo.svg' },
      items: [
        { type: 'docSidebar', sidebarId: 'docs', position: 'left', label: 'Docs' },
        { to: '/blog', label: 'Changelog', position: 'left' },
        { type: 'docsVersionDropdown', position: 'right' },
        { type: 'localeDropdown', position: 'right' },
        { href: 'https://github.com/myorg/my-project', label: 'GitHub', position: 'right' },
      ],
    },
    prism: {
      theme: prismThemes.github,
      darkTheme: prismThemes.dracula,
      additionalLanguages: ['bash', 'json', 'yaml', 'typescript'],
    },
    // Algolia DocSearch (see below)
    algolia: {
      appId: 'YOUR_APP_ID',
      apiKey: 'YOUR_SEARCH_API_KEY',
      indexName: 'YOUR_INDEX_NAME',
    },
  },
};

export default config;
```

## MDX: Interactive Documentation

Docusaurus uses MDX, letting you embed React components in Markdown:

```mdx
---
title: Quick Start
description: Get up and running in 5 minutes
---

import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';
import CodeBlock from '@theme/CodeBlock';

# Quick Start

Install the SDK for your language:

<Tabs groupId="programming-language">
  <TabItem value="ts" label="TypeScript" default>
    ```bash
    npm install @myproject/sdk
    ```
  </TabItem>
  <TabItem value="python" label="Python">
    ```bash
    pip install myproject-sdk
    ```
  </TabItem>
  <TabItem value="go" label="Go">
    ```bash
    go get github.com/myorg/myproject-sdk
    ```
  </TabItem>
</Tabs>

import { ApiExample } from '@site/src/components/ApiExample';

<ApiExample endpoint="/api/users" method="GET" />
```

## Custom React Components in Docs

```typescript
// src/components/ApiExample.tsx
import React from 'react';
import CodeBlock from '@theme/CodeBlock';

interface ApiExampleProps {
  endpoint: string;
  method: 'GET' | 'POST' | 'PUT' | 'DELETE';
  requestBody?: object;
  response?: object;
}

export function ApiExample({ endpoint, method, requestBody, response }: ApiExampleProps) {
  const curlCommand = `curl -X ${method} https://api.myproject.com${endpoint} \\
  -H "Authorization: Bearer YOUR_API_KEY" \\
  -H "Content-Type: application/json"${
    requestBody ? ` \\\n  -d '${JSON.stringify(requestBody, null, 2)}'` : ''
  }`;

  return (
    <div className="api-example">
      <div className="api-example__method">{method}</div>
      <div className="api-example__endpoint">{endpoint}</div>
      <CodeBlock language="bash">{curlCommand}</CodeBlock>
      {response && (
        <details>
          <summary>Response</summary>
          <CodeBlock language="json">{JSON.stringify(response, null, 2)}</CodeBlock>
        </details>
      )}
    </div>
  );
}
```

## Versioning

```bash
# Snapshot the current docs as version 2.0
npm run docusaurus docs:version 2.0
```

This creates `versioned_docs/version-2.0/` and adds `2.0` to `versions.json`. Users can switch between versions via the dropdown.

```typescript
// docusaurus.config.ts — custom version labels
docs: {
  versions: {
    current: { label: '3.x (Next)', path: 'next' },
    '2.0': { label: '2.x (Stable)', path: '2.0' },
    '1.x': { label: '1.x (Legacy)', path: '1.x', banner: 'unmaintained' },
  },
}
```

## Algolia DocSearch Setup

Algolia DocSearch is free for open-source projects. For private docs, use Algolia's paid tier.

**Step 1: Apply for DocSearch** at docsearch.algolia.com (open-source) or create an Algolia account.

**Step 2: Add the config** (Docusaurus automatically integrates):

```typescript
themeConfig: {
  algolia: {
    appId: 'B1G2GM9NG0',
    apiKey: 'a1b2c3d4e5f6',  // Public search key (safe to expose)
    indexName: 'my-project-docs',
    contextualSearch: true,   // Filter by current version/locale
    searchParameters: {
      facetFilters: ['language:en', 'version:2.0'],
    },
  },
}
```

**Step 3: Configure the Algolia crawler** (`algolia.config.json`):

```json
{
  "index_name": "my-project-docs",
  "start_urls": ["https://docs.myproject.com"],
  "sitemap_urls": ["https://docs.myproject.com/sitemap.xml"],
  "selectors": {
    "lvl0": ".menu__link--sublist-caret",
    "lvl1": "article h1",
    "lvl2": "article h2",
    "lvl3": "article h3",
    "text": "article p, article li"
  },
  "custom_settings": {
    "attributesForFaceting": ["language", "version", "type"]
  }
}
```

## Deployment

**Vercel** (zero-config):

```bash
# Install Vercel CLI
npm i -g vercel

# Deploy
vercel --prod
```

**GitHub Pages**:

```yaml
# .github/workflows/docs.yml
name: Deploy Docs
on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0  # Full git history for timestamps
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - run: npm ci
      - run: npm run build
      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./build
```

## Docusaurus vs Alternatives

| | Docusaurus | VitePress | Mintlify | GitBook |
|--|-----------|-----------|---------|--------|
| React support | Full | No | Partial | No |
| MDX | Native | No | No | No |
| Versioning | Built-in | Manual | Built-in | Built-in |
| Custom components | Anything | Vue only | Limited | No |
| Self-hosted | Yes | Yes | No | Optional |
| Best for | Complex OSS docs | Simple Vue projects | API docs | Non-technical teams |

Docusaurus is the best choice when you need: versioning, i18n, custom React components, or you're maintaining an open-source project with evolving APIs.
