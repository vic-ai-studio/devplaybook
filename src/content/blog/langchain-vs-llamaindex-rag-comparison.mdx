Both LangChain and LlamaIndex help you build RAG (Retrieval-Augmented Generation) applications, but they come from different philosophies. Choosing the wrong one leads to fighting the framework instead of building your product. Here's the real difference.

## Philosophy Difference

**LangChain** is a general-purpose LLM application framework. It gives you composable building blocks—chains, agents, tools, memory—that can be assembled into any LLM-powered workflow. RAG is one use case among many.

**LlamaIndex** was built specifically for RAG. Its entire API is optimized around indexing documents, querying them intelligently, and getting grounded answers from LLMs. If you're building a document Q&A system, LlamaIndex speaks that language natively.

The question is: are you building a RAG system, or an LLM-powered application that uses RAG as one component?

## Architecture Comparison

### LangChain's RAG Pipeline

LangChain's RAG is built by connecting components in a chain:

```python
from langchain_openai import ChatOpenAI, OpenAIEmbeddings
from langchain_community.vectorstores import Chroma
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_core.runnables import RunnablePassthrough
from langchain_community.document_loaders import PyPDFLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter

# 1. Load and split documents
loader = PyPDFLoader("docs/handbook.pdf")
docs = loader.load()

splitter = RecursiveCharacterTextSplitter(chunk_size=1000, chunk_overlap=200)
chunks = splitter.split_documents(docs)

# 2. Create vector store
embeddings = OpenAIEmbeddings()
vectorstore = Chroma.from_documents(chunks, embeddings)
retriever = vectorstore.as_retriever(search_kwargs={"k": 4})

# 3. Build the RAG chain
prompt = ChatPromptTemplate.from_template("""
Answer the question based only on the context below.

Context: {context}
Question: {question}
""")

llm = ChatOpenAI(model="gpt-4o", temperature=0)

def format_docs(docs):
    return "\n\n".join(doc.page_content for doc in docs)

rag_chain = (
    {"context": retriever | format_docs, "question": RunnablePassthrough()}
    | prompt
    | llm
    | StrOutputParser()
)

# 4. Query
answer = rag_chain.invoke("What is the vacation policy?")
print(answer)
```

### LlamaIndex's RAG Pipeline

LlamaIndex handles the same workflow with higher-level abstractions:

```python
from llama_index.core import VectorStoreIndex, SimpleDirectoryReader, Settings
from llama_index.llms.openai import OpenAI
from llama_index.embeddings.openai import OpenAIEmbedding

# Configure models once
Settings.llm = OpenAI(model="gpt-4o", temperature=0)
Settings.embed_model = OpenAIEmbedding(model="text-embedding-3-small")

# 1. Load documents (from a directory)
documents = SimpleDirectoryReader("docs/").load_data()

# 2. Index them (handles chunking and embedding automatically)
index = VectorStoreIndex.from_documents(documents)

# 3. Query engine — that's it
query_engine = index.as_query_engine(similarity_top_k=4)

response = query_engine.query("What is the vacation policy?")
print(response.response)

# Access retrieved source nodes
for node in response.source_nodes:
    print(f"Score: {node.score:.3f} | {node.text[:200]}")
```

LlamaIndex does more in fewer lines for pure RAG use cases.

## Vector Stores

Both integrate with all major vector databases:

```python
# LangChain with Pinecone
from langchain_pinecone import PineconeVectorStore

vectorstore = PineconeVectorStore.from_documents(
    chunks,
    embeddings,
    index_name="my-index"
)

# LlamaIndex with Pinecone
from llama_index.vector_stores.pinecone import PineconeVectorStore
from llama_index.core import StorageContext

vector_store = PineconeVectorStore(index_name="my-index")
storage_context = StorageContext.from_defaults(vector_store=vector_store)
index = VectorStoreIndex.from_documents(
    documents,
    storage_context=storage_context
)
```

## Query Engines: LlamaIndex's Superpower

LlamaIndex's query engines go far beyond basic similarity search:

```python
from llama_index.core.query_engine import (
    SubQuestionQueryEngine,
    RouterQueryEngine,
)
from llama_index.core.tools import QueryEngineTool

# Sub-question engine: breaks complex questions into sub-queries
products_engine = products_index.as_query_engine()
reviews_engine = reviews_index.as_query_engine()

tools = [
    QueryEngineTool.from_defaults(query_engine=products_engine, description="Product catalog"),
    QueryEngineTool.from_defaults(query_engine=reviews_engine, description="Customer reviews"),
]

sub_question_engine = SubQuestionQueryEngine.from_defaults(query_engine_tools=tools)

response = sub_question_engine.query(
    "What are the top products and what do customers say about them?"
)
# Automatically queries both indexes and synthesizes the answer
```

LangChain achieves similar results but requires more manual chain construction.

## When to Use LangChain

LangChain excels when RAG is **one part of a larger system**:

```python
from langchain.agents import AgentExecutor, create_openai_tools_agent
from langchain_core.tools import tool

@tool
def search_docs(query: str) -> str:
    """Search internal documentation."""
    return rag_chain.invoke(query)

@tool
def query_database(sql: str) -> str:
    """Query the production database."""
    return db.execute(sql)

@tool
def send_slack_message(channel: str, message: str) -> str:
    """Send a Slack notification."""
    return slack.chat_postMessage(channel=channel, text=message)

# Agent that uses RAG as one of many tools
agent = create_openai_tools_agent(llm, [search_docs, query_database, send_slack_message], prompt)
agent_executor = AgentExecutor(agent=agent, tools=[...])

agent_executor.invoke({"input": "Check the docs for our refund policy and notify #support"})
```

LangChain also has better support for:
- Streaming responses
- Conversation memory and history management
- Complex multi-step agent workflows
- LangSmith for tracing and debugging

## When to Use LlamaIndex

LlamaIndex excels for **document-heavy RAG applications**:

- Enterprise document Q&A (PDFs, wikis, knowledge bases)
- Multi-index querying with routing logic
- Structured data extraction from documents
- Citation-backed answers where source tracing matters

```python
# LlamaIndex excels at advanced retrieval strategies
from llama_index.core.retrievers import VectorIndexRetriever, BM25Retriever
from llama_index.core.retrievers import QueryFusionRetriever

# Hybrid retrieval: vector similarity + keyword search
vector_retriever = VectorIndexRetriever(index=index, similarity_top_k=5)
bm25_retriever = BM25Retriever.from_defaults(index=index, similarity_top_k=5)

hybrid_retriever = QueryFusionRetriever(
    retrievers=[vector_retriever, bm25_retriever],
    num_queries=3,  # generate query variations
    mode="reciprocal_rerank",
)

query_engine = RetrieverQueryEngine(retriever=hybrid_retriever)
```

## Performance Comparison

For a 100-document RAG benchmark (ingestion + query):

| Metric | LangChain | LlamaIndex |
|---|---|---|
| Lines of code (basic RAG) | ~30 | ~15 |
| Ingestion flexibility | High | High |
| Query sophistication (out of box) | Medium | High |
| Agent/tool integration | Excellent | Good |
| Streaming support | Excellent | Good |
| Community size | Larger | Focused |
| Learning curve | Steeper | Gentler for RAG |

## The Verdict

**Use LlamaIndex when** your app is primarily about querying documents, knowledge bases, or structured data. The query engines, node postprocessors, and retrieval strategies are purpose-built for RAG and require less glue code.

**Use LangChain when** you're building agents, chatbots, or workflows where RAG is one capability among many. LangChain's composability and tool/agent ecosystem are unmatched.

Many production systems use both: LlamaIndex for the retrieval layer, LangChain for the orchestration layer. They integrate cleanly:

```python
from langchain_core.tools import tool
from llama_index.core import VectorStoreIndex

index = VectorStoreIndex.from_documents(documents)
query_engine = index.as_query_engine()

@tool
def search_knowledge_base(query: str) -> str:
    """Search the company knowledge base."""
    response = query_engine.query(query)
    return str(response)
# Use this tool inside a LangChain agent
```

Don't feel forced to choose. Use each framework where it's strongest.
