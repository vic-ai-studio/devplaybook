# Web3 Development in 2026: Tools, Frameworks, and Architecture Patterns

The Web3 tooling landscape has matured significantly. You no longer need to fight your tools — the right stack lets you build production-grade dApps with the same confidence as Web2. This guide covers the current best practices for serious Web3 development.

---

## The Modern Web3 Stack

A production Web3 application typically has four layers:

1. **Smart contracts** — Solidity on EVM chains (or Rust/Anchor for Solana)
2. **Development environment** — Hardhat or Foundry for compile/test/deploy
3. **Frontend library** — viem + wagmi for type-safe chain interaction
4. **Indexing layer** — The Graph or a custom indexer for historical data

---

## ethers.js vs viem: The Shift is Real

ethers.js dominated 2019–2023. viem is now the preferred choice for new projects:

```typescript
// ethers.js v6
import { ethers } from 'ethers';
const provider = new ethers.BrowserProvider(window.ethereum);
const signer = await provider.getSigner();
const contract = new ethers.Contract(address, abi, signer);
const tx = await contract.transfer(to, amount);
await tx.wait();
```

```typescript
// viem — fully typed, tree-shakeable, no class bloat
import { createWalletClient, custom, parseUnits } from 'viem';
import { mainnet } from 'viem/chains';

const client = createWalletClient({
  chain: mainnet,
  transport: custom(window.ethereum!),
});

const hash = await client.writeContract({
  address: '0x...',
  abi: erc20Abi,
  functionName: 'transfer',
  args: [to, parseUnits('1.0', 18)],
});
```

**Why viem wins:**
- Tree-shakeable — bundle only what you use
- Full TypeScript inference from ABI (with `const abi = [...] as const`)
- No BigNumber wrapper — native BigInt
- 10-20x smaller bundle for equivalent functionality
- Built-in ABIType codegen via [wagmi CLI](https://wagmi.sh/cli)

## Hardhat vs Foundry

Both are excellent. The choice depends on your workflow:

| Aspect | Hardhat | Foundry |
|---|---|---|
| Language | JavaScript/TypeScript | Solidity (tests in Sol) |
| Speed | Good | 10-100x faster |
| Ecosystem | Huge plugin ecosystem | Growing |
| Fuzzing | Plugin required | Built-in (forge fuzz) |
| Fork testing | ✅ | ✅ |
| Debugging | Good stack traces | Verbose traces |

### Foundry Setup (recommended for new projects)

```bash
curl -L https://foundry.paradigm.xyz | bash
```

```solidity
// test/Token.t.sol — tests are Solidity
pragma solidity ^0.8.20;

import "forge-std/Test.sol";
import "../src/Token.sol";

contract TokenTest is Test {
    Token token;
    address alice = makeAddr("alice");
    address bob = makeAddr("bob");

    function setUp() public {
        token = new Token("Test", "TST", 1_000_000e18);
        token.transfer(alice, 100e18);
    }

    function test_Transfer() public {
        vm.prank(alice);
        token.transfer(bob, 10e18);
        assertEq(token.balanceOf(bob), 10e18);
    }

    function testFuzz_Transfer(uint256 amount) public {
        amount = bound(amount, 1, token.balanceOf(alice));
        vm.prank(alice);
        token.transfer(bob, amount);
        assertEq(token.balanceOf(alice), 100e18 - amount);
    }
}
```

## wagmi v2: React Hooks for Web3

wagmi v2 is a complete rewrite built on viem and TanStack Query:

```bash
npm install wagmi viem @tanstack/react-query
```

```typescript
// app.tsx — setup
import { WagmiProvider, createConfig, http } from 'wagmi';
import { mainnet, arbitrum, base } from 'wagmi/chains';
import { injected, metaMask, coinbaseWallet } from 'wagmi/connectors';
import { QueryClient, QueryClientProvider } from '@tanstack/react-query';

const config = createConfig({
  chains: [mainnet, arbitrum, base],
  connectors: [injected(), metaMask(), coinbaseWallet({ appName: 'My App' })],
  transports: {
    [mainnet.id]: http('https://eth-mainnet.g.alchemy.com/v2/KEY'),
    [arbitrum.id]: http('https://arb-mainnet.g.alchemy.com/v2/KEY'),
    [base.id]: http('https://base-mainnet.g.alchemy.com/v2/KEY'),
  },
});

const queryClient = new QueryClient();

export default function App() {
  return (
    <WagmiProvider config={config}>
      <QueryClientProvider client={queryClient}>
        <MyApp />
      </QueryClientProvider>
    </WagmiProvider>
  );
}
```

```typescript
// components/TokenBalance.tsx
import { useReadContract, useAccount } from 'wagmi';
import { erc20Abi } from 'viem';
import { formatUnits } from 'viem';

export function TokenBalance({ tokenAddress }: { tokenAddress: `0x${string}` }) {
  const { address } = useAccount();

  const { data: balance, isLoading } = useReadContract({
    address: tokenAddress,
    abi: erc20Abi,
    functionName: 'balanceOf',
    args: [address!],
    query: { enabled: !!address },
  });

  const { data: decimals } = useReadContract({
    address: tokenAddress,
    abi: erc20Abi,
    functionName: 'decimals',
  });

  if (isLoading) return <span>Loading…</span>;
  if (!balance || !decimals) return null;
  return <span>{formatUnits(balance, decimals)} tokens</span>;
}
```

```typescript
// Writing to contracts with optimistic updates
import { useWriteContract, useWaitForTransactionReceipt } from 'wagmi';

export function MintButton() {
  const { writeContract, data: hash, isPending } = useWriteContract();
  const { isLoading: isConfirming, isSuccess } = useWaitForTransactionReceipt({ hash });

  return (
    <button
      onClick={() => writeContract({
        address: NFT_ADDRESS,
        abi: nftAbi,
        functionName: 'mint',
        args: [1n],
        value: parseEther('0.05'),
      })}
      disabled={isPending || isConfirming}
    >
      {isPending ? 'Awaiting wallet…' : isConfirming ? 'Confirming…' : isSuccess ? 'Minted!' : 'Mint NFT'}
    </button>
  );
}
```

## IPFS for Decentralized Storage

For NFT metadata and assets, IPFS via Pinata is the standard:

```typescript
// Upload file to IPFS via Pinata API
async function uploadToIPFS(file: File): Promise<string> {
  const formData = new FormData();
  formData.append('file', file);
  formData.append('pinataMetadata', JSON.stringify({ name: file.name }));

  const response = await fetch('https://api.pinata.cloud/pinning/pinFileToIPFS', {
    method: 'POST',
    headers: { Authorization: `Bearer ${process.env.PINATA_JWT}` },
    body: formData,
  });

  const data = await response.json();
  return `ipfs://${data.IpfsHash}`;
}

// Upload metadata JSON
async function uploadMetadata(metadata: object): Promise<string> {
  const response = await fetch('https://api.pinata.cloud/pinning/pinJSONToIPFS', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      Authorization: `Bearer ${process.env.PINATA_JWT}`,
    },
    body: JSON.stringify({ pinataContent: metadata }),
  });

  const data = await response.json();
  return `ipfs://${data.IpfsHash}`;
}
```

## The Graph: Indexing On-Chain Data

Direct RPC calls can't efficiently query historical data. The Graph indexes events and makes them queryable via GraphQL:

```graphql
# schema.graphql — define your data model
type Transfer @entity {
  id: ID!
  from: Bytes!
  to: Bytes!
  value: BigInt!
  timestamp: BigInt!
  blockNumber: BigInt!
}

type User @entity {
  id: ID! # wallet address
  sentTransfers: [Transfer!]! @derivedFrom(field: "from")
  receivedTransfers: [Transfer!]! @derivedFrom(field: "to")
  totalReceived: BigInt!
}
```

```typescript
// subgraph.yaml — map events to entities
dataSources:
  - kind: ethereum/contract
    name: Token
    network: mainnet
    source:
      address: "0x..."
      abi: Token
      startBlock: 18000000
    mapping:
      kind: ethereum/events
      apiVersion: 0.0.7
      language: wasm/assemblyscript
      entities:
        - Transfer
        - User
      eventHandlers:
        - event: Transfer(indexed address,indexed address,uint256)
          handler: handleTransfer
```

```typescript
// Query from frontend
const TRANSFERS_QUERY = `
  query GetRecentTransfers($first: Int!) {
    transfers(first: $first, orderBy: timestamp, orderDirection: desc) {
      id
      from
      to
      value
      timestamp
    }
  }
`;

const response = await fetch('https://api.thegraph.com/subgraphs/name/org/token', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ query: TRANSFERS_QUERY, variables: { first: 10 } }),
});
```

## Security Checklist Before Mainnet

- [ ] Full test coverage with Foundry fuzz tests
- [ ] Slither static analysis: `slither . --config-file slither.config.json`
- [ ] Reentrancy guards on all external calls (`nonReentrant` or CEI pattern)
- [ ] No `tx.origin` for auth — use `msg.sender`
- [ ] Oracle manipulation resistance (TWAP not spot price)
- [ ] Overflow checks (Solidity 0.8+ has built-in, but custom assembly doesn't)
- [ ] Access control on admin functions (Ownable or AccessControl)
- [ ] External audit for contracts holding >$100K
