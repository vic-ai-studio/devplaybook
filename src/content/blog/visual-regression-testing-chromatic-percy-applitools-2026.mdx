# Visual Regression Testing: Chromatic vs Percy vs Applitools 2026

Visual regression testing catches unintended UI changes by comparing screenshots between commits. Without it, a CSS change that accidentally breaks a button layout ships to production undetected. The three main tools — Chromatic, Percy, and Applitools — take different approaches to the same problem.

## How Visual Regression Testing Works

1. **Baseline**: First run captures screenshots as the "correct" state
2. **Comparison**: Each subsequent run compares new screenshots to baselines
3. **Review**: Differences surface for human review — accept if intentional, reject if a bug
4. **Updated baseline**: Accepted changes become the new baseline

The challenge is distinguishing **intentional changes** (you redesigned a button) from **regressions** (you changed a color variable and it broke 12 components).

## Chromatic

Built by the Storybook team, Chromatic is tightly integrated with Storybook. It captures each story as a visual test, making it ideal for component-driven development.

**Setup**:

```bash
npm install --save-dev chromatic
```

```yaml
# GitHub Actions
- name: Publish to Chromatic
  uses: chromaui/action@v11
  with:
    projectToken: ${{ secrets.CHROMATIC_PROJECT_TOKEN }}
    exitZeroOnChanges: true  # Don't fail CI, just report changes
```

**Key Features**:

- **Turbosnap**: Only re-tests stories affected by changed files (using git diff + dependency graph)
- **Component-level diffs**: Changes are shown at the story level, not just page level
- **Auto-accept**: Accept changes from a specific branch automatically (e.g., your design branch)
- **Interaction tests**: Run Storybook play functions before taking screenshots

```typescript
// A story with interaction test
export const FilledForm: Story = {
  play: async ({ canvasElement }) => {
    const canvas = within(canvasElement);
    await userEvent.type(canvas.getByRole('textbox', { name: /email/i }), 'test@test.com');
    await userEvent.click(canvas.getByRole('button', { name: /submit/i }));
  },
};
// Chromatic: takes screenshot AFTER the play function runs
```

**Pricing**: Free tier (5,000 snapshots/month); paid from $149/month.

**Best for**: Teams using Storybook for component development.

## Percy (BrowserStack)

Percy captures full-page and component screenshots integrated with E2E test frameworks (Cypress, Playwright, Selenium). Now part of BrowserStack.

**Playwright Integration**:

```typescript
import { test, expect } from '@playwright/test';
import { percySnapshot } from '@percy/playwright';

test('product listing page', async ({ page }) => {
  await page.goto('https://shop.example.com/products');
  await page.waitForLoadState('networkidle');

  // Take a Percy snapshot
  await percySnapshot(page, 'Product Listing');

  // Test interactions
  await page.click('[data-testid="filter-electronics"]');
  await percySnapshot(page, 'Product Listing - Electronics Filter');
});
```

**Cypress Integration**:

```javascript
cy.visit('/dashboard');
cy.percySnapshot('Dashboard - Default State');

cy.get('[data-testid="dark-mode-toggle"]').click();
cy.percySnapshot('Dashboard - Dark Mode');
```

**Key Features**:

- Works with any test framework that can run in a browser
- Responsive snapshots: test multiple viewport widths in one snapshot
- Smart diff: ignores non-visual changes (timestamps, dynamic content)
- BrowserStack integration for cross-browser visual testing

**Pricing**: Free tier (5,000 snapshots/month); paid from $299/month.

**Best for**: Teams with existing Cypress or Playwright E2E suites who want to add visual checks without switching to Storybook.

## Applitools Eyes

The most sophisticated option — uses AI to distinguish meaningful visual differences from irrelevant ones (animations, dynamic content, font rendering differences).

```typescript
// Playwright + Applitools
import { test } from '@playwright/test';
import { Eyes, Target, VisualGridRunner } from '@applitools/eyes-playwright';

test('visual check checkout flow', async ({ page }) => {
  const runner = new VisualGridRunner({ testConcurrency: 5 });
  const eyes = new Eyes(runner);

  await eyes.open(page, 'E-commerce App', 'Checkout Flow', {
    width: 1280,
    height: 768,
  });

  await page.goto('https://shop.example.com/checkout');

  // Layout check: tolerates minor positional differences
  await eyes.check('Checkout Form', Target.window().layout());

  // Strict check: pixel-perfect for hero images
  await eyes.check('Product Image', Target.region(productImageLocator).strict());

  await eyes.close();
});
```

**Key Differentiators**:

- **Visual AI**: Trained on billions of images to ignore irrelevant diffs (anti-aliasing, browser rendering differences) while catching real regressions
- **Ultrafast Grid**: Renders on real browsers in parallel — cross-browser visual testing at scale
- **Root Cause Analysis**: Pinpoints the CSS property that changed and shows which elements are affected
- **PDF comparison**: Compare documents and reports

**Pricing**: Free tier (limited); Team from $299/month; Enterprise custom.

**Best for**: Enterprise teams with cross-browser requirements and complex dynamic UIs.

## Choosing the Right Tool

**Choose Chromatic** if:
- Your team uses Storybook
- You want component-level visual testing
- You want the best Storybook integration with CI

**Choose Percy** if:
- You already have Cypress or Playwright tests
- You want to add visual assertions without changing your testing framework
- You want full-page screenshots from E2E tests

**Choose Applitools** if:
- You need cross-browser visual testing at scale
- Your UI has lots of dynamic content that causes false positives
- You have a large QA team and need advanced features (root cause analysis, PDF testing)

## Reducing False Positives

All tools struggle with dynamic content (timestamps, animations, dynamic data). Strategies:

```typescript
// Playwright: hide dynamic elements before snapshot
await page.evaluate(() => {
  document.querySelectorAll('[data-testid="timestamp"]')
    .forEach(el => el.style.visibility = 'hidden');
});
await percySnapshot(page, 'Dashboard');

// Or use data-testid to freeze animations
await page.addStyleTag({ content: '*, *::before, *::after { animation: none !important; }' });
```

Visual regression testing is high-value for design systems and component libraries. For feature-rich applications, focus it on critical paths (checkout, onboarding) rather than trying to cover every page.
