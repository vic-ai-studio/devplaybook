# pnpm Workspaces: The Complete Monorepo Guide 2026

pnpm workspaces is the foundation layer for JavaScript monorepos in 2026. Unlike npm/yarn workspaces, pnpm's symlink-free, content-addressable store eliminates phantom dependencies and dramatically reduces `node_modules` size. Combined with Turborepo or Nx, it's the standard stack for multi-package JavaScript repositories.

## Why pnpm Over npm/yarn Workspaces

```bash
# npm workspaces: hoisting creates phantom dependencies
# Package A can import Package B's dependencies even without declaring them
# → Works locally, fails in production

# pnpm: strict isolation
# Package A can ONLY import what it declares in package.json
# → Prevents phantom deps, catches missing deps early
```

Storage efficiency: pnpm's content-addressable store means all packages sharing a dependency version store it once globally:

```
# 10 projects, each with React 18 as dependency
# npm: 10 copies of React 18 (10 × 120MB = 1.2GB)
# pnpm: 1 copy of React 18 (120MB) — hardlinked to each project
```

## Workspace Protocol: Internal Dependencies

The `workspace:` protocol links internal packages — no version number required:

```json
// apps/web/package.json
{
  "name": "@myorg/web",
  "dependencies": {
    "@myorg/ui": "workspace:*",     // Always latest local version
    "@myorg/utils": "workspace:^",  // Semver-compatible local version
    "@myorg/types": "workspace:*",
    "next": "15.0.0",
    "react": "18.3.0"
  }
}
```

When you run `pnpm install`, these workspace packages are linked as symlinks in `node_modules`. When you publish packages (with `pnpm publish`), the `workspace:*` is replaced with the actual published version.

## pnpm Commands

```bash
# Install all packages
pnpm install

# Add dependency to a specific workspace
pnpm add react --filter @myorg/web
pnpm add -D typescript --filter @myorg/api

# Add dependency to ALL workspaces
pnpm add -D prettier -r --filter="*"

# Run script in all workspaces
pnpm -r run build

# Run script in specific workspace
pnpm --filter @myorg/web run dev

# Run script in workspace and its dependencies
pnpm --filter @myorg/web... run build

# List all workspace packages
pnpm list -r --depth 0 --json

# Check for outdated packages
pnpm outdated -r
```

## Publishing Packages

For public packages from a monorepo:

```bash
# Changeset workflow (recommended)
pnpm add -Dw @changesets/cli
npx changeset init

# Create a changeset (when making changes)
npx changeset  # Interactive: select packages + version bump

# Version packages
npx changeset version  # Bumps versions, updates CHANGELOG.md

# Publish
pnpm build -r  # Build all packages first
npx changeset publish  # Publishes changed packages to npm
```

## CI Pipeline

```yaml
# .github/workflows/ci.yml
name: CI
on: [push, pull_request]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0  # Needed for turbo --filter with git comparison

      - uses: pnpm/action-setup@v3
        with:
          version: 9

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'pnpm'

      - run: pnpm install --frozen-lockfile

      - uses: actions/cache@v4
        with:
          path: .turbo
          key: ${{ runner.os }}-turbo-${{ github.sha }}
          restore-keys: ${{ runner.os }}-turbo-

      - run: pnpm turbo run build test lint --filter=...[HEAD^1]
```
