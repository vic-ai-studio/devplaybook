# IoT Development in 2026: Protocols, Platforms, and Edge Computing

IoT is no longer just hobby projects and Arduino LEDs. Production IoT systems handle billions of messages per day, run inference at the edge, and require the same engineering rigor as backend services. This guide covers modern IoT development from protocol selection to fleet management.

---

## Protocol Selection: MQTT vs HTTP vs CoAP vs WebSocket

Choosing the wrong protocol wastes battery, bandwidth, and development time.

| Protocol | Transport | Overhead | Best For |
|---|---|---|---|
| MQTT | TCP | Very low (2-byte header) | Constrained devices, telemetry |
| HTTP/1.1 | TCP | High (headers ~200-800 bytes) | Commands, config APIs |
| CoAP | UDP | Very low (4-byte header) | Very constrained devices (motes) |
| WebSocket | TCP | Low | Bidirectional streaming |
| AMQP | TCP | Medium | Enterprise message queuing |

### MQTT: The IoT Workhorse

MQTT's publish-subscribe model, QoS levels, and retained messages make it ideal for telemetry:

```python
# Device side — Paho MQTT Python
import paho.mqtt.client as mqtt
import json
import time

client = mqtt.Client(client_id="device-001", protocol=mqtt.MQTTv5)

# TLS with client certificates (production requirement)
client.tls_set(
    ca_certs='/certs/AmazonRootCA1.pem',
    certfile='/certs/device.crt',
    keyfile='/certs/device.key',
)

def on_connect(client, userdata, flags, rc, properties):
    print(f"Connected: {rc}")
    # Subscribe to command topic
    client.subscribe("devices/device-001/commands", qos=1)

def on_message(client, userdata, msg):
    command = json.loads(msg.payload)
    if command.get('action') == 'reboot':
        schedule_reboot()

client.on_connect = on_connect
client.on_message = on_message
client.connect("your-iot-endpoint.amazonaws.com", 8883)
client.loop_start()

# Publish telemetry every 30 seconds
while True:
    payload = {
        "temperature": read_temperature(),
        "humidity": read_humidity(),
        "timestamp": int(time.time()),
        "device_id": "device-001",
    }
    # QoS 1: at least once delivery
    client.publish("devices/device-001/telemetry", json.dumps(payload), qos=1)
    time.sleep(30)
```

**QoS levels:**
- QoS 0: Fire and forget (best for frequent, lossy-tolerant data like sensor readings)
- QoS 1: At least once (ensures delivery, possible duplicates — use idempotent processing)
- QoS 2: Exactly once (guaranteed, highest overhead — use for commands, payments)

## AWS IoT Core

AWS IoT Core is the dominant managed MQTT broker for production:

```python
# AWS IoT Thing provisioning (one-time setup)
import boto3

iot = boto3.client('iot', region_name='us-east-1')

# Create thing
thing = iot.create_thing(thingName='device-001', thingTypeName='TemperatureSensor')

# Create and attach policy
policy = iot.create_policy(
    policyName='DeviceTelemetryPolicy',
    policyDocument=json.dumps({
        "Version": "2012-10-17",
        "Statement": [{
            "Effect": "Allow",
            "Action": ["iot:Connect", "iot:Publish", "iot:Subscribe", "iot:Receive"],
            "Resource": [
                f"arn:aws:iot:us-east-1:ACCOUNT_ID:client/${{iot:ClientId}}",
                f"arn:aws:iot:us-east-1:ACCOUNT_ID:topic/devices/${{iot:ClientId}}/*",
                f"arn:aws:iot:us-east-1:ACCOUNT_ID:topicfilter/devices/${{iot:ClientId}}/*",
            ]
        }]
    })
)

# Create certificate
cert = iot.create_keys_and_certificate(setAsActive=True)
iot.attach_policy(policyName='DeviceTelemetryPolicy', target=cert['certificateArn'])
iot.attach_thing_principal(thingName='device-001', principal=cert['certificateArn'])
```

## Azure IoT Hub

Azure IoT Hub is preferred in enterprise environments with existing Azure infrastructure:

```python
# Azure IoT Hub — device SDK
from azure.iot.device import IoTHubDeviceClient, Message
from azure.iot.device import MethodResponse
import json

CONNECTION_STRING = "HostName=myhub.azure-devices.net;DeviceId=device-001;SharedAccessKey=KEY"
client = IoTHubDeviceClient.create_from_connection_string(CONNECTION_STRING)

# Handle direct methods (commands from cloud)
def method_handler(method_request):
    if method_request.name == "reboot":
        payload = {"result": "rebooting", "delay": method_request.payload.get("delay", 5)}
        response = MethodResponse.create_from_method_request(method_request, 200, payload)
    else:
        response = MethodResponse.create_from_method_request(method_request, 404, {})
    client.send_method_response(response)

client.on_method_request_received = method_handler
client.connect()

# Send telemetry
message = Message(json.dumps({"temp": 23.5, "hum": 65}))
message.content_type = "application/json"
message.content_encoding = "utf-8"
message.custom_properties["device_type"] = "temperature_sensor"
client.send_message(message)
```

## Edge Computing with WASM

Running inference and logic on-device reduces latency and bandwidth:

```rust
// Rust → WASM for edge inference
use wasm_bindgen::prelude::*;

#[wasm_bindgen]
pub fn classify_anomaly(temperature: f32, humidity: f32, pressure: f32) -> u8 {
    // Simple threshold-based anomaly detection
    // In production: run ONNX model or TFLite
    let score = (temperature - 25.0).abs() * 0.4
              + (humidity - 60.0).abs() * 0.3
              + (pressure - 1013.0).abs() * 0.003;

    if score > 20.0 { 2 }      // Critical
    else if score > 10.0 { 1 } // Warning
    else { 0 }                  // Normal
}
```

```bash
# Build WASM for embedded (no-std)
cargo build --target wasm32-unknown-unknown --release

# Or use WasmEdge for server-side edge nodes
wasmedge target/wasm32-unknown-unknown/release/sensor.wasm
```

## OTA (Over-The-Air) Updates

Never ship a device that can't be updated — firmware bugs will require recalls otherwise:

```python
# AWS IoT Jobs for OTA
import boto3, hashlib

iot = boto3.client('iot')

# Create OTA job
job = iot.create_job(
    jobId='firmware-v2.1.3',
    targets=['arn:aws:iot:us-east-1:ACCOUNT:thinggroup/production-sensors'],
    document=json.dumps({
        "operation": "update_firmware",
        "version": "2.1.3",
        "url": "https://firmware.example.com/v2.1.3/sensor.bin",
        "checksum": hashlib.sha256(firmware_bytes).hexdigest(),
        "size": len(firmware_bytes),
    }),
    jobExecutionsRetryConfig={"criteriaList": [{"failureType": "FAILED", "numberOfRetries": 3}]},
    presignedUrlConfig={"roleArn": "arn:aws:iam::ACCOUNT:role/OTARole", "expiresInSec": 3600},
)
```

```c
// Device-side OTA handler (ESP-IDF)
#include "esp_ota_ops.h"
#include "esp_https_ota.h"

void ota_task(void* pvParam) {
    esp_http_client_config_t config = {
        .url = firmware_url,
        .cert_pem = server_cert,
        .timeout_ms = 5000,
    };

    esp_https_ota_config_t ota_config = {
        .http_config = &config,
        .http_client_init_cb = NULL,
        .bulk_flash_erase = false,
    };

    esp_err_t ret = esp_https_ota(&ota_config);
    if (ret == ESP_OK) {
        esp_restart(); // Boot new firmware
    } else {
        // Rollback to previous firmware
        esp_ota_mark_app_invalid_rollback_and_reboot();
    }
}
```

## IoT Security Essentials

IoT devices are perennial security weak points. Non-negotiables:

```python
# 1. Device identity: unique certificate per device (not shared credentials)
# Never do this:
SHARED_DEVICE_KEY = "aaaabbbbcccc"  # BAD — one compromise = all devices compromised

# Do this: each device gets its own cert during manufacturing
# 2. TLS everywhere — no plaintext MQTT on port 1883 in production

# 3. Firmware signing — verify before flashing
import hashlib, hmac

def verify_firmware(firmware_bytes: bytes, signature: bytes, public_key: bytes) -> bool:
    from cryptography.hazmat.primitives.asymmetric import ec, utils
    from cryptography.hazmat.primitives import hashes
    from cryptography.hazmat.primitives.serialization import load_pem_public_key

    key = load_pem_public_key(public_key)
    try:
        key.verify(signature, firmware_bytes, ec.ECDSA(hashes.SHA256()))
        return True
    except Exception:
        return False

# 4. Secure element for key storage (not flash)
# Use ATECC608A or similar hardware secure element
# Keys never leave the chip — only signing operations

# 5. Regular rotation of device credentials
# 6. Network segmentation — IoT VLAN separate from corporate network
```

## Fleet Management at Scale

```python
# AWS IoT Device Shadow — sync desired/reported state
import boto3

iot_data = boto3.client('iot-data', region_name='us-east-1')

# Update desired state (from cloud → device)
iot_data.update_thing_shadow(
    thingName='device-001',
    payload=json.dumps({
        "state": {
            "desired": {
                "sampling_interval": 60,
                "alert_threshold": 85.0,
                "led_enabled": True,
            }
        }
    })
)

# Device reads and reports actual state
shadow = json.loads(iot_data.get_thing_shadow(thingName='device-001')['payload'].read())
desired = shadow['state'].get('desired', {})
reported = shadow['state'].get('reported', {})

# Apply desired settings
apply_config(desired)

# Report actual state back
iot_data.update_thing_shadow(
    thingName='device-001',
    payload=json.dumps({
        "state": {
            "reported": {
                "sampling_interval": desired.get('sampling_interval', 30),
                "firmware_version": "2.1.3",
                "uptime_seconds": get_uptime(),
            }
        }
    })
)
```

## Quick Architecture Reference

```
Device Layer
    │ MQTT (TLS, QoS 1)
    ▼
IoT Broker (AWS IoT Core / Azure IoT Hub)
    │ Rules Engine
    ├──► Time Series DB (Timestream / InfluxDB)
    ├──► Stream Processing (Kinesis / Event Hub)
    ├──► Lambda/Functions (alerts, commands)
    └──► S3/Blob (raw data lake)

Edge Layer (optional)
    Greengrass / IoT Edge
    ├──► Local MQTT broker
    ├──► WASM/TFLite inference
    └──► 5-minute local buffering for connectivity loss
```
