## DW-3: Error Message Rewrite

Rewrite these technical error messages to be user-friendly.

Errors: {{PASTE_ERROR_MESSAGES}}

Product: {{DESCRIBE_PRODUCT}}. Users: {{TECHNICAL_or_NON_TECHNICAL}}

For each error: (1) user-friendly message, (2) what caused it (plain language), (3) what to do next, (4) whether to show a support link.

Never say: "Error", "Invalid", "Failed", "Exception". Always tell the user what to do next.
